# パス名処理方針

## 基本方針

XER では、`std::filesystem::path` は用いず、独自の `path` クラスを提供する。  
パス名処理は UTF-8 ベースで統一し、内部表現の単純性と一貫性を優先する。

## `path` クラス

### 内部表現

- `path` は内部で `std::u8string` を保持する。
- 内部表現におけるディレクトリ区切り文字は常に `/` とする。
- コンストラクタでは入力中の `\` を `/` に正規化する。
- Windows における先頭部分の意味は保持する。

### 先頭部分の扱い

Windows では、区切り文字は `/` に統一してよいが、先頭部分の意味は保持する。  
たとえば以下は相互に区別される。

- `C:foo`
- `C:/foo`
- `/foo`
- `//server/share/foo`

### 文字列取得

- `path` は `str()` を持つ。
- `str()` の返却値は `std::u8string_view` とする。
- `str()` は内部正規形の文字列を返す。

### 表示用文字列

必要に応じて、人間向け表示のために `/` を `\` に置換した文字列を生成する関数を用意してよい。  
ただし、内部表現そのものは `/` のままとする。

## メンバ関数とフリー関数

XER では、メンバ関数にする必然がないものはフリー関数として提供する。  
これにより、`path` の責務を内部表現の保持と不変条件の維持に限定し、カプセル化を強固にする。

### メンバ関数

- コンストラクタ
- `str()`
- `operator/=`

### フリー関数

- `operator/`
- `basename`
- `parent_path`
- `extension`
- `stem`
- `is_absolute`
- `is_relative`

## パス結合

### `operator/=`

- `operator/=` は `path` のメンバ関数とする。
- 左辺を書き換える基本操作として扱う。

### `operator/`

- `operator/` はフリー関数とする。
- `operator/=` を用いて実装する。

## Windows における絶対パス判定

Windows では以下のように扱う。

- `X:foo` は絶対パスではない
- `X:/foo` は絶対パスである
- `/foo` は絶対パスである
- `//server/share/foo` は絶対パスである

## Windows におけるパス結合規則

### 基本規則

- 右辺が通常の相対パスであれば連結する。
- 右辺が同一ドライブのドライブ相対パスであれば連結する。
- 右辺が異なるドライブのドライブ相対パスであれば結合不能とし、エラーとする。
- 原則として、右辺が絶対パスである場合、結合は不能としエラーとする。

### 例外規則

ただし、以下の 2 つは例外として許可する。

- `/` + `/a` = `/a`
- `X:` + `/a` = `X:/a`

### 具体例

- `C:/a` + `b` = `C:/a/b`
- `C:/a` + `C:b` = `C:/a/b`
- `C:/a` + `D:b` = エラー

## 字面的な親パス

`parent_path` は字面的な親パスを返すフリー関数とする。  
これは実在ファイルシステムに依存しない純粋な字面操作である。

### 規則

- 先頭部分を保ったまま、最後の要素を 1 つ取り除く。
- これ以上字面的に親を取れない場合はエラーとする。

### 例

- `a/b/c` -> `a/b`
- `a/b` -> `a`
- `a` -> エラー

- `/a/b` -> `/a`
- `/a` -> `/`
- `/` -> エラー

- `X:a/b` -> `X:a`
- `X:a` -> `X:`
- `X:` -> エラー

- `X:/a/b` -> `X:/a`
- `X:/a` -> `X:/`
- `X:/` -> エラー

## 実在ファイルシステム依存の処理

以下のような処理は `path` の字面処理とは分離し、別途関数として提供する。

- 相対パスの解決
- 絶対化
- シンボリックリンクの解決
- 実在ファイルシステムを参照する正規化

## `basename`

### 基本方針

- `basename` はフリー関数とする。
- 引数は `path` 型のみを受け取る。
- 戻り値は `std::u8string_view` とする。
- 末尾の `/` の扱いは PHP の `basename` と同様とする。

### PHP との非互換

以下の点で PHP の `basename` とは非互換とする。

- ロケール非依存
- `\` を区切り文字として扱わない
- 普通の文字列は受け取らず、`path` 型のみを受け取る

### 区切り文字

- `basename` が区切り文字として扱うのは `/` のみである。
- ただし、`path` のコンストラクタで `\` は `/` に正規化される。

## `extension`

### 基本方針

- `extension` はフリー関数とする。
- 戻り値は `std::u8string_view` とする。
- 対象は `basename(path)` とする。

### 定義

- `basename(path)` に含まれる最初の `.` 以降の部分を返す。
- `.` が存在しない場合は空文字列を返す。
- 先頭の `.` も特別扱いしない。

### 例

- `c.txt` -> `.txt`
- `archive.tar.gz` -> `.tar.gz`
- `.foo` -> `.foo`
- `foo.` -> `.`
- `foo` -> 空文字列
- `.foo.bar` -> `.foo.bar`

### 開始位置指定

- `extension` は追加引数として開始位置を取れるようにしてよい。
- この開始位置は `basename` に対する位置とする。
- 指定位置以降で最初に見つかった `.` から末尾までを返す。
- 見つからなければ空文字列を返す。

## `stem`

### 基本方針

- `stem` はフリー関数とする。
- 戻り値は `std::u8string_view` とする。

### 定義

- `stem(path)` は `basename(path)` から `extension(path)` を除いた前半を返す。

### 例

- `c.txt` -> `c`
- `archive.tar.gz` -> `archive`
- `.foo` -> 空文字列
- `foo.` -> `foo`
- `foo` -> `foo`

## まとめ

- `path` は UTF-8 の `std::u8string` を内部保持する。
- 内部区切り文字は `/` に統一する。
- コンストラクタで `\` を `/` に正規化する。
- 先頭部分の Windows 的意味は保持する。
- `str()` は `std::u8string_view` を返す。
- `/=` はメンバ関数、`/` はフリー関数とする。
- 分解・判定系は原則フリー関数とする。
- `parent_path` は字面的な親のみを扱う。
- 実在ファイルシステム依存の解決処理は別関数とする。
- `basename` は PHP 互換寄りだが、ロケール非依存・`\` 非対応・`path` 専用とする。
- `extension` は `basename` 中の最初の `.` 以降を返す。
- `stem` は `basename` から `extension` を除いた前半を返す。
