# stdlib.h におけるマルチバイト文字変換の方針

## 概要

XER における `stdlib.h` のマルチバイト文字変換機能は、C 標準ライブラリのロケール依存機構をそのまま再現するのではなく、XER 全体の文字コード方針に合わせて再設計する。

対象とする文字エンコーディングは、以下の 4 種類に限定する。

- CP932
- UTF-8
- UTF-16
- UTF-32

エスケープシーケンスやシフト状態を持つ状態依存エンコーディングは扱わない。

---

## 基本方針

XER におけるマルチバイト文字変換では、内部の基準文字型として `char32_t` および `char16_t` を用いる。  
`char` および `wchar_t` は、既存ライブラリやプラットフォーム API との接点として扱うが、内部基準にはしない。

`char` の既定解釈はプラットフォームごとに定める。

- Windows では既定で CP932
- Linux では既定で UTF-8

必要に応じて、この既定値はマクロにより切り替えられるようにしてよい。

`wchar_t` は既存ライブラリとの接点として扱う。

- Windows では実質 UTF-16 側の窓口
- Linux では実質 UTF-32 側の窓口

ただし、いずれの場合も `wchar_t` 自体を XER の内部基準にはしない。

多バイト側の明示的な入出力型には、以下を用いる。

- `char`
- `unsigned char`
- `char8_t`

それぞれの意味は次の通りとする。

- `char`
  - 既存ライブラリ接点用
  - 解釈は OS 既定または明示設定に従う
- `unsigned char`
  - CP932 固定
- `char8_t`
  - UTF-8 固定

変換先または変換元の文字型には、以下を用いる。

- `wchar_t`
- `char16_t`
- `char32_t`

---

## 関数体系

単一文字を扱う関数として、以下を提供する。

- `mblen`
- `mbtotc`
- `tctomb`

文字列を扱う関数として、以下を提供する。

- `mbstotcs`
- `tcstombs`

ここで、各名前の意味は次の通りとする。

- `mb`
  - multibyte
- `tc`
  - text character
- `tcs`
  - text character sequence

`tc` は、以下の文字型を総称する抽象名である。

- `wchar_t`
- `char16_t`
- `char32_t`

C 標準ライブラリにおける `mbtowc`, `wctomb`, `mbstowcs`, `wcstombs` に相当する機能は、上記の名称体系にまとめて提供する。

---

## オーバーロード方針

これらの関数はテンプレートではなく、オーバーロード集合として提供する。

`mbtotc` は、多バイト列から 1 文字を取り出す関数である。  
入力側として、以下を受理する。

- `const char*`
- `const unsigned char*`
- `const char8_t*`

出力側として、以下を受理する。

- `wchar_t*`
- `char16_t*`
- `char32_t*`

`tctomb` は、1 文字を多バイト列へ変換する関数である。  
入力側として、以下を受理する。

- `wchar_t`
- `char16_t`
- `char32_t`

出力側として、以下を受理する。

- `char*`
- `unsigned char*`
- `char8_t*`

`mbstotcs` は、多バイト文字列を文字列へ変換する関数である。  
入力側として、以下を受理する。

- `const char*`
- `const unsigned char*`
- `const char8_t*`

出力側として、以下を受理する。

- `wchar_t*`
- `char16_t*`
- `char32_t*`

`tcstombs` は、文字列を多バイト文字列へ変換する関数である。  
入力側として、以下を受理する。

- `const wchar_t*`
- `const char16_t*`
- `const char32_t*`

出力側として、以下を受理する。

- `char*`
- `unsigned char*`
- `char8_t*`

---

## 状態付き変換

C 標準ライブラリにおける `mbrlen`, `mbrtowc`, `wcrtomb`, `mbsrtowcs`, `wcsrtombs` に相当する専用関数名は設けない。

代わりに、`mblen`, `mbtotc`, `tctomb`, `mbstotcs`, `tcstombs` に対して、`xer::mbstate_t*` を追加引数として受けるオーバーロードを用意する。これにより、状態付き変換を表現する。

状態引数を受けないオーバーロードは、毎回独立した変換を行う。  
状態引数を受けるオーバーロードは、`xer::mbstate_t` に保持された状態を利用して変換を継続する。

内部静的状態は持たない。

---

## `xer::mbstate_t`

`xer::mbstate_t` は、状態付き変換のための状態を保持する型である。

この型は少なくとも、以下の情報を保持する。

- エンコーディング種別
- 未完了バイト列
- 保持中バイト数

未完了バイト列を保持する以上、その列がどのエンコーディングとして解釈されるべきかも保持しなければならない。そのため、状態オブジェクトにはエンコーディング種別を持たせる。

`char` を受けた場合も、状態へ格納する時点で実際のエンコーディング種別を確定させる。たとえば、Windows 上の `char` 入力は CP932、Linux 上の `char` 入力は UTF-8 として状態に保持する。

`xer::mbstate_t` にはデフォルトコンストラクタを定義し、未使用状態を生成できるようにする。

`reset()` のような再初期化メンバは設けない。  
初期状態へ戻したい場合は、新しい `xer::mbstate_t` を作成する。

---

## 返り値

これらの関数の返り値型は、原則として以下とする。

```cpp
xer::result<std::size_t>
```

`xer::result<std::size_t>` の実体は `std::expected<std::size_t, error<void>>` であり、XER における標準的な失敗表現として用いる。

成功時の意味は C 標準ライブラリにおける対応関数に概ね準じるが、失敗時は特殊値ではなくエラーで表す。

---

## 返り値の意味

### `mblen`

- 成功時は 1 文字を構成するバイト数を返す
- 空文字列を指している場合は 0 を返す
- 不正な入力や未完成入力は失敗を返す

### `mbtotc`

- 成功時は読み取ったバイト数を返す
- 変換結果の 1 文字は出力先へ格納する
- 空文字列を指している場合は 0 を返す
- 不正な入力や未完成入力は失敗を返す

### `tctomb`

- 成功時は出力したバイト数を返す
- 変換不能文字は失敗を返す

### `mbstotcs`

- 成功時は変換した文字数を返す
- 出力先が `nullptr` の場合は、変換後の文字数だけを数えて返してよい
- 不正な入力が含まれる場合は失敗を返す

### `tcstombs`

- 成功時は変換したバイト数を返す
- 出力先が `nullptr` の場合は、変換後のバイト数だけを数えて返してよい
- 変換不能文字が含まれる場合は失敗を返す

---

## エラー方針

これらの関数は通常の失敗を `xer::result` により表現する。

主に想定する失敗原因は以下のとおりである。

- 不正なマルチバイト列
- 途中で終わっている不完全なマルチバイト列
- 変換不能文字
- 引数不正
- 状態オブジェクトと入力エンコーディングの不整合

エラー型の詳細化が不要な段階では、`error<void>` を用いてよい。

必要に応じて、将来的には不正位置や原因種別などを `error<Detail>` により詳細化してもよい。

---

## `char` の既定解釈

`char` 入出力は、明示指定がない限り OS ごとの既定エンコーディングで解釈する。

- Windows: CP932
- Linux: UTF-8

ただし、これは XER の内部標準表現ではなく、あくまで既存 API との接点としての解釈である。

内部処理は最終的に UTF-16 / UTF-32 / UTF-8 のいずれかの明確な表現へ正規化して扱う。

---

## 実装方針

### 低水準変換の利用

単一文字変換の内部実装では、文字コード変換方針で定めた低水準変換関数を利用してよい。

たとえば、以下のような流れを許容する。

- `unsigned char*` / `char*` / `char8_t*` の入力を 1 文字ぶん読み出す
- 必要に応じて UTF-32 へ正規化する
- 出力型に応じて `wchar_t`, `char16_t`, `char32_t` へ格納する

### 文字列変換の構築

文字列変換関数 `mbstotcs`, `tcstombs` は、単一文字変換を繰り返す形で構築してよい。

このとき、状態付き版では `mbstate_t` を引き回しながら処理する。

---

## 対象外

初期段階では、少なくとも以下は対象外とする。

- ロケール依存変換
- シフト状態を持つエンコーディング
- 内部静的状態を前提とした C 互換 API
- `char` 入力に対する高度な自動判定
- UTF-7 や ISO-2022-JP などの特殊エンコーディング

---

## 補足

XER におけるマルチバイト文字変換は、C 標準ライブラリとの名前上の対応を保ちつつも、設計上は XER 全体の文字コード方針およびエラー処理方針に従って再設計する。

したがって、互換性よりも次の点を優先する。

- 対象エンコーディングの明確化
- ロケール非依存
- 内部表現の明確化
- 特殊値ではなく `xer::result` による失敗表現
