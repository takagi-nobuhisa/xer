# PHP を用いたコードジェネレーションとテスト方針

## 概要

XER では、コード生成およびテスト補助のために PHP を使用する。  
想定用途は、変換表の生成、テストケースの生成、テストプログラムのビルド・実行制御、結果集計などである。

PHP は Web アプリケーション開発用言語として認識されることが多いが、本プロジェクトでは汎用のスクリプト言語として利用する。  
特に、C プログラマにとって文法が比較的馴染みやすいことを重視する。

PHP は XER の開発時に使用する補助ツールであり、XER の利用者に PHP の使用を要求しない。  
XER を利用するだけであれば、通常の C++ コンパイラとビルド環境があればよい。

リポジトリには PHP スクリプトおよびそれに関連する生成用データを含める。  
一方、ライブラリ単体の配布物には、通常これらを含めない。

---

## PHP の役割

PHP は、XER 本体の一部ではなく、開発時専用ツールとして使用する。  
用途は主として以下である。

- 変換表の生成
- テストケースの生成
- テストプログラムの生成
- テストプログラムのビルド・実行制御
- テスト結果の集計

---

## テストの分類

XER のテストは、少なくとも以下に分類する。

- 正常系コンパイルテスト
- 異常系コンパイルテスト
- 実行テスト

### 正常系コンパイルテスト

正常系コンパイルテストでは、正しく使用されたコードがコンパイルできることを確認する。  
公開ヘッダの組み合わせテストもこれに含む。

### 異常系コンパイルテスト

異常系コンパイルテストでは、意図された誤用が期待通りにコンパイルエラーとなることを確認する。  
この種のテストでは、エラー理由を安定して判定できるようにするため、主として `static_assert` によるエラー検出を用いる。

テストの合格条件は、コンパイルが失敗することに加え、想定した `static_assert` のメッセージが診断出力に含まれることである。

テストケースの定義方法や、ソースコード中へのメタデータ埋め込み方法については、実装時に別途定める。

### 実行テスト

実行テストは、XER 独自の `assert` 系マクロを用いて記述する。  
`assert` 系マクロは、標準の `assert` のように異常終了させるのではなく、失敗時に XER 専用の例外型を送出する。

実行テストで対応する検査は、少なくとも以下を含む。

- 真偽判定
- 否定判定
- 2 値比較
- 大小比較（当面は `<` のみ）
- 例外送出確認
- 例外非送出確認

## 公開ヘッダの配置

XER の公開ヘッダは `xer/` ディレクトリ直下に配置する。  
`xer/` のサブディレクトリ以下にあるヘッダは公開ヘッダとはみなさず、実装詳細または内部補助用とする。

公開ヘッダかどうかは拡張子ではなく配置場所によって決定する。  
したがって、公開ヘッダの一覧は `xer/` ディレクトリ直下のファイル一覧から自動抽出する。  
このとき、サブディレクトリ以下は対象に含めない。

---

## 公開ヘッダの組み合わせコンパイルテスト

公開ヘッダについては、インクルード順序に依存したコンパイルエラーを防ぐため、正常系コンパイルテストとして、異なる 2 個の公開ヘッダの順序付き組み合わせテストを行う。

公開ヘッダが `n` 個ある場合、2 ヘッダの順序付き組み合わせは `nP2` 通りであるため、これらすべてについてテスト用ソースファイルを PHP により自動生成し、コンパイルを行う。

なお、公開ヘッダ単独でのインクルード可能性については、通常の実行テストがそれを兼ねるものとし、専用の単独インクルードテストは原則として設けない。  
ただし、各公開ヘッダは少なくとも 1 つの通常テストにおいて単独でインクルードされることを前提とする。

## assert 系マクロ

### 位置づけ

XER の `assert` 系マクロは、ライブラリ内部の実行テストだけでなく、XER の利用者にも公開してよい。

ただし、`assert` 系マクロは主として XER 自身の開発および簡易テスト用途を想定するものであり、あらゆる型・出力環境・文字列表現に対して完全な利便性を保証するものではない。

利用上の制約や、値レポートに関する制限事項は、ドキュメントおよびコメントで明示するものとする。  
ライブラリ側では、必要以上の過剰なケアは行わない。

### マクロ名

実行テスト用の `assert` 系マクロ名は、`xer_` をプレフィックスとして、少なくとも以下を用意する。

- `xer_assert`
- `xer_assert_not`
- `xer_assert_eq`
- `xer_assert_ne`
- `xer_assert_lt`
- `xer_assert_throw`
- `xer_assert_nothrow`
