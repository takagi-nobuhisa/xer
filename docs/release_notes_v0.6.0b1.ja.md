# xer v0.6.0b1 リリースノート

## 概要

v0.6.0b1 は v0.6.0 系の最初のベータ版です。

このベータ版では、主に次の 2 つの機能領域を追加しています。

- `<xer/zip.h>` による ZIP アーカイブ操作
- `<xer/serialize.h>` と PHP 製 `xfer` 生成スクリプトによる固定スキーマのバイナリシリアライズ

また、逐次入力 API における EOF の扱いも整理しました。

---

## ZIP アーカイブ操作

`<xer/zip.h>` は、ZIP を圧縮・アーカイブ操作のための機能として提供します。

v0.6.0b1 での対象範囲は次のとおりです。

- ZIP アーカイブを開く
- エントリメタ情報の逐次読み取り
- エントリ名による完全一致検索
- エントリ全体の読み取り
- ZIP アーカイブの作成
- メモリ上のバイト列を deflate 圧縮して追加
- ファイルを deflate 圧縮して追加
- `zip_commit` による明示的な書き込み確定
- 単一エントリを指定ファイルへ展開
- 単一エントリを展開先ディレクトリ配下へ展開
- 全エントリを展開先ディレクトリ配下へ展開

初期実装の対象は、通常の非 ZIP64・単一ディスク ZIP アーカイブです。ZIP64、暗号化エントリ、コメント、分割 ZIP、大きなエントリのストリーミング I/O は今後の課題としています。

## 固定スキーマのバイナリシリアライズ

`<xer/serialize.h>` は、次の低水準バイナリアーカイブを提供します。

- `xer::binary_output_archive`
- `xer::binary_input_archive`

この形式は自己記述型ではありません。固定されたフィールド順序とリトルエンディアンのスカラ表現で、データだけを保存します。

低水準で対応するフィールド型は次のとおりです。

- `bool`
- 固定幅の符号付き整数・符号なし整数
- `float`
- `double`
- `std::u8string`
- `std::vector<std::byte>`
- `std::array<T, N>`
- `std::vector<T>`
- `std::map<K, V>`

ユーザー定義構造体は、リフレクションではなく、生成された `xfer` 関数で扱います。

---

## PHP `xfer` 生成スクリプト

`php/generate_xfer_struct.php` は、PHP のスキーマ配列から C++ の構造体と `xfer` 関数を生成します。

対応内容は次のとおりです。

- 単一構造体スキーマ
- 複数構造体スキーマ
- `u32`、`s`、`bin`、`v`、`m`、`[a, N]` などの短い型トークン
- 生成日時を表すスキーマタイムスタンプ定数
- 再現可能な生成例・テストのための `--generated-at=<text>`
- 使用するフィールド型に応じた include の最小化
- 識別子、名前空間、対応型式の検証

## EOF の共通化

逐次入力 API では、次のデータを読み取ろうとして入力が尽きている場合に `error_t::end_of_file` を返します。

これにより、名前やキーによる検索失敗を表す `error_t::not_found` と EOF を区別します。

例:

- `zip_read` は、これ以上エントリがない場合に `end_of_file` を返します。
- `zip_locate_name` は、指定名のエントリがない場合に `not_found` を返します。
- ストリームやディレクトリの逐次読み取りも、入力終端を `end_of_file` として扱います。
