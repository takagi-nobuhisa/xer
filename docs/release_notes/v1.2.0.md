# xer C++ Utility Library v1.2.0 Release Notes

xer C++ Utility Library v1.2.0 adds modular interoperation support, reorganizes the diagnostic API, reduces test time, and improves portability across C++ implementations.

This release adds `<xer/dlfcn.h>` as a new public header for loading shared libraries at run time. It also renames the former `<xer/diag.h>` header to `<xer/diagnostics.h>` and organizes simple printing, tracing, and logging as diagnostic facilities.

The public-header combination test now uses one generated include-order source per public header by default instead of compiling every ordered two-header pair. This preserves practical include-order checking while significantly reducing normal test time.

## Highlights

- Added the new public header `<xer/dlfcn.h>`.
- Renamed `<xer/diag.h>` to `<xer/diagnostics.h>`.
- Added `xer_assert_throw_any` to `<xer/assert.h>`.
- Clarified that `xer_assert_throw` catches the specified exception type by `const` reference.
- Reduced the cost of public-header include-order testing.
- Retained the legacy full ordered-pair test through `--full` and `--all-pairs`.
- Made aliases that use non-standard identifier characters opt-in.
- Established portable ASCII unit aliases: `m2`, `m3`, `sec2`, and `sec3`.
- Updated README files, the public-header list, policy documents, and both reference manuals.

## New Public Header

### `<xer/dlfcn.h>`

A small cross-platform API has been added for loading shared libraries and resolving symbols at run time.

The main types and functions are:

```cpp
auto dlopen(std::string_view path) -> result<shared_library>;
auto dlclose(shared_library& library) noexcept -> void;
auto is_open(const shared_library& library) noexcept -> bool;
auto native_handle(const shared_library& library) noexcept -> void*;

auto dlsym(
    const shared_library& library,
    std::string_view name,
    void*& out) -> result<void>;

template<class Function>
auto dlsym(
    const shared_library& library,
    std::string_view name,
    Function*& out) -> result<void>;
```

`shared_library` is a copyable shared handle. Copies refer to the same native library handle, and the library is unloaded when the last reference is released.

The implementation wraps `dlopen`, `dlsym`, and `dlclose` on POSIX systems, and `LoadLibraryW`, `GetProcAddress`, and `FreeLibrary` on Windows.

The function-pointer overload of `dlsym` infers the function type from the output pointer, so callers do not need to write an explicit cast.

The initial implementation uses `RTLD_NOW | RTLD_LOCAL` on POSIX systems and `LoadLibraryW` on Windows. Platform-specific loading flags are not exposed yet.

## Diagnostic Header Rename

The former public header `<xer/diag.h>` has been renamed to `<xer/diagnostics.h>`.

The new name more clearly describes its role as the common public header for simple printing, tracing, and logging.

The main facilities include:

```cpp
xer_print(expression)
xer_print(label, expression)
xer_trace(category, level, object)
xer_log(category, level, message)
xer_log(category, level, format, ...)
```

`xer_trace` is disabled when `NDEBUG` is defined. `xer_log` is not disabled merely by `NDEBUG`; it can be disabled at compile time by defining `XER_ENABLE_LOG` to `0`.

### Compatibility Note

`<xer/diag.h>` is no longer a public header. Existing code must use:

```cpp
#include <xer/diagnostics.h>
```

## Assertion Updates

`xer_assert_throw_any` has been added to `<xer/assert.h>`.

```cpp
xer_assert_throw_any(expr)
```

This macro verifies that evaluating an expression throws some exception. It is useful when a test does not require a specific exception type or when the thrown value is not a standard exception object.

The existing `xer_assert_throw` macro checks a specific exception type and catches it by `const` reference.

```cpp
xer_assert_throw(expr, exception_type)
```

If the expression does not throw, or throws an exception that does not match the specified type, the assertion reports failure through `xer::assertion_error`.

## Public-Header Include-Order Tests

The default strategy of `php/test_public_header_pairs.php` has been changed.

Previously, with `n` public headers, the script generated and compiled every ordered pair of different headers, for a total of `nP2` sources.

In v1.2.0, the default strategy generates one source for each public header. Each generated source includes the target header first and then includes all other available public headers.

With 47 public headers, the default strategy compiles 47 generated sources. This reduces test time while still checking that:

- each public header can be included first
- each public header provides the declarations required for its own use
- later public headers do not expose an include-order dependency

The legacy full ordered-pair strategy remains available:

```sh
php test_public_header_pairs.php --full
php test_public_header_pairs.php --all-pairs
```

`--all` disables incremental filtering and reruns the complete set selected by the current strategy.

## Opt-In Non-Standard Identifiers

Aliases that contain characters not guaranteed to be valid in standard C++ identifiers, such as superscript digits, are no longer provided by default.

Portable ASCII unit aliases are provided as:

```cpp
m2
m3
sec2
sec3
```

Aliases containing superscript digits are available only when `XER_ENABLE_NON_STANDARD_IDENTIFIERS` is defined before including the header:

```cpp
#define XER_ENABLE_NON_STANDARD_IDENTIFIERS
#include <xer/quantity.h>
```

This enables aliases such as:

```cpp
m²
m³
sec²
sec³
```

The same policy applies to non-standard Unicode aliases for mathematical and physical constants.

Portable code, especially code compiled with options such as `-pedantic`, should use the ASCII identifiers.

## Modular Interoperation Policy

A new policy document defines the direction for dynamic shared libraries, delayed loading, shared memory, synchronization primitives, message queues, and possible future RPC and language-binding support.

v1.2.0 implements `shared_library` and `<xer/dlfcn.h>` as the first independent building block in that direction.

The policy separates the ordinary C++ API from any future externally published C ABI. The C++ API may use `xer::result<T>`, RAII, and templates, while an external ABI should use C-compatible types, status codes, output parameters, and opaque handles.

## Compatibility Notes

v1.2.0 contains the following source-compatibility changes:

- `<xer/diag.h>` was renamed to `<xer/diagnostics.h>`.
- Non-standard identifiers such as `m²`, `m³`, `sec²`, and `sec³` are not provided unless `XER_ENABLE_NON_STANDARD_IDENTIFIERS` is defined.
- Portable unit identifiers should use `m2`, `m3`, `sec2`, and `sec3`.

`<xer/dlfcn.h>` and `xer_assert_throw_any` are additive features and do not replace existing APIs.
