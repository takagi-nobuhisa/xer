# xer C++ Utility Library v1.0.0 Release Notes

xer C++ Utility Library v1.0.0 is the first stable release of xer.

This release focuses on stabilizing the public API, cleaning up long-standing inconsistencies, strengthening formatted and stream-based I/O for xer value types, improving tests and examples, and bringing the documentation set to a more complete state.

The main goal of v1.0.0 is not to add a large new feature area. Instead, it establishes the current design as a stable foundation for future development.

## Highlights

- Declared the current public API surface as the v1.0.0 stable baseline.
- Reviewed public API naming, return-value policy, and error-handling policy.
- Added `error_t::success` as the explicit success value for `error_t`.
- Strengthened `%@` formatting and xer value-type stream output.
- Added stream input support for additional xer value types where appropriate.
- Implemented `mod` and `umod` consistently with the existing division helpers.
- Removed the old `from_radian` and `to_radian` cyclic aliases; use `from_rad` and `to_rad`.
- Added tests and examples for `parse`, `version`, color conversion, cyclic/interval concepts, and `%@` formatting.
- Added PlantUML-based diagrams to key documentation areas.
- Completed and refreshed the Japanese reference fragments for v1.0.0.
- Updated README files, public header lists, reference manual fragments, examples, and policy documents.

## Stable Public API Baseline

v1.0.0 establishes the current set of public headers as the stable baseline for the library.

The public header set includes practical utilities for:

- arithmetic and binary helpers
- string, encoding, Unicode, and Japanese text processing
- path, process, stream, and file I/O
- INI, JSON, TOML, ZIP, and serialization support
- image manipulation and drawing
- math, matrix, cyclic angle values, intervals, quantities, and statistics
- MeCab-based Japanese text processing
- diagnostics, assertions, scope guards, and type information

The public header list has been checked against the actual files under `xer/*.h`.

## API Review and Cleanup

### `error_t::success`

`error_t::success` was added as the explicit success value.

This removes the need to treat `static_cast<error_t>(0)` as a special case in error-name and error-message handling.

Updated areas include:

- `strerror(error_t::success)`
- `get_error_name(error_t::success)`
- `get_errno_name(error_t::success)`
- stream output and input for `error_t`
- related tests and documentation

### `mod` and `umod`

`<xer/arithmetic.h>` now has consistent modulo helpers.

`mod` provides signed modulo handling in the same family as the existing division helpers.

`umod` is the unsigned-domain modulo helper and follows the same error-reporting style as `udiv`.

The behavior is aligned with the existing division policy:

- divide by zero is reported as `error_t::divide_by_zero`
- unsigned-domain misuse is reported as `error_t::dom`
- `xer::result` operands propagate errors

### Cyclic radian aliases

The older cyclic helper names:

```cpp
from_radian
to_radian
```

were removed.

Use the shorter names instead:

```cpp
from_rad
to_rad
```

This keeps the angle-conversion API consistent with `from_degree`, `to_degree`, and the rest of the cyclic angle policy.

## Formatted I/O and Stream I/O

v1.0.0 improves xer value-type I/O.

The `%@` formatter is strengthened as a practical way to format xer value types through the library's formatted-output layer.

Additional iostream operators were added or improved for value types such as:

- `matrix`
- `vec`
- `polar`
- color value types
- image geometry types
- cyclic and interval values
- paths and quantities

This makes diagnostics, examples, and simple tools easier to write.

A new example demonstrates `%@` formatting together with ordinary iostream extraction:

```text
examples/example_iostream_percent_at.cpp
```

## Tests and Examples

v1.0.0 adds and strengthens tests around areas that are easy to break by hand editing or API cleanup.

Notable additions include:

- `tests/test_parse.cpp`
- `tests/test_version.cpp`
- `examples/example_parse_basic.cpp`
- `examples/example_color_conversion.cpp`
- `examples/example_cyclic_interval_concepts.cpp`
- `examples/example_iostream_percent_at.cpp`

`tests/test_version.cpp` checks consistency between the preprocessor macros and namespace constants in `<xer/version.h>`.
It also checks that `version_string` is constructed from `major.minor.patch + suffix`.

The examples were also reviewed so that documentation references point to actual files.

## Documentation Improvements

v1.0.0 significantly improves the documentation set.

New PlantUML-based diagrams were added for:

- image coordinates and stride model
- MeCab processing pipeline
- color conversion relationships
- cyclic and interval concepts

The diagrams are stored as `.puml` sources and generated as PNG images under `docs/images`.

The relevant policy documents and reference fragments now refer to these diagrams.

MeCab documentation was strengthened with:

- API selection guidance
- layer-by-layer usage explanation
- existing example-program mapping
- common usage patterns
- error behavior by layer

The Japanese reference fragments were updated for v1.0.0, and the generated Japanese reference manual no longer contains untranslated-section markers.

## Compatibility Notes

### Source compatibility

v1.0.0 is intended as the stable baseline.

Most changes are additive or documentation/test improvements.

The main source-level cleanup to note is the removal of:

```cpp
from_radian
to_radian
```

Use:

```cpp
from_rad
to_rad
```

instead.

### Error handling

Adding `error_t::success` should make error handling clearer and more explicit.
Code that previously assumed `static_cast<error_t>(0)` as a special unnamed success value should migrate to `error_t::success`.

## Summary

v1.0.0 marks xer as a usable stable foundation.

The library now has a broad but coherent set of utility headers, a clearer error model, more consistent formatted and stream-based I/O, stronger tests around version and parsing behavior, and more complete English and Japanese documentation.

Future releases can now build on this baseline with a clearer distinction between stable public APIs and experimental additions.
