# xer C++ Utility Library v1.0.0 リリースノート

xer C++ Utility Library v1.0.0 は、xer の最初の安定版リリースです。

このリリースでは、公開 API の安定化、長く残っていた不整合の整理、xer の値型に対するフォーマット出力およびストリーム入出力の強化、テストとコード例の補強、ドキュメント全体の整備を中心に行いました。

v1.0.0 の主目的は、大きな新機能領域を追加することではありません。現在の設計を、今後の開発の安定した土台として確定することにあります。

## ハイライト

- 現在の公開 API 群を v1.0.0 の安定版ベースラインとして整理
- 公開 API の命名、戻り値方針、エラー処理方針を確認
- `error_t` に明示的な成功値 `error_t::success` を追加
- `%@` フォーマットと xer 値型のストリーム出力を強化
- 必要な xer 値型に対してストリーム入力を追加・整理
- 既存の除算ヘルパーと整合する `mod` / `umod` を実装
- 古い cyclic 変換別名 `from_radian` / `to_radian` を削除し、`from_rad` / `to_rad` に整理
- `parse`、`version`、色変換、cyclic/interval 概念、`%@` フォーマットのテストとコード例を追加
- 主要ドキュメントに PlantUML ベースの図を追加
- v1.0.0 に合わせて日本語リファレンス断片を更新
- README、公開ヘッダ一覧、リファレンスマニュアル断片、コード例、方針文書を更新

## 安定版公開 API ベースライン

v1.0.0 では、現在の公開ヘッダ群をライブラリの安定版ベースラインとして位置付けます。

公開ヘッダは、次のような実用的な機能領域を含みます。

- 算術・バイナリ補助
- 文字列、エンコーディング、Unicode、日本語処理
- パス、プロセス、ストリーム、ファイル I/O
- INI、JSON、TOML、ZIP、シリアライズ
- 画像処理と描画
- 数学、行列、cyclic 角度値、interval、物理量、統計
- MeCab ベースの日本語解析
- 診断出力、アサーション、スコープガード、型情報

公開ヘッダ一覧は、実際の `xer/*.h` と照合済みです。

## API の確認と整理

### `error_t::success`

`error_t` に明示的な成功値として `error_t::success` を追加しました。

これにより、`static_cast<error_t>(0)` を名前のない特別値として扱う必要がなくなります。

更新対象には次が含まれます。

- `strerror(error_t::success)`
- `get_error_name(error_t::success)`
- `get_errno_name(error_t::success)`
- `error_t` のストリーム出力・入力
- 関連テストとドキュメント

### `mod` と `umod`

`<xer/arithmetic.h>` に、既存の除算ヘルパーと整合する剰余ヘルパーを追加しました。

`mod` は符号付き剰余を扱うヘルパーです。

`umod` は符号無し領域向けの剰余ヘルパーで、`udiv` と同じエラー報告方針に従います。

挙動は既存の除算方針に合わせています。

- ゼロ除算は `error_t::divide_by_zero`
- 符号無し領域に合わない使い方は `error_t::dom`
- `xer::result` オペランドのエラーは伝播

### cyclic のラジアン変換別名

古い cyclic ヘルパー名である次の別名を削除しました。

```cpp
from_radian
to_radian
```

今後は短い名前を使用します。

```cpp
from_rad
to_rad
```

これにより、`from_degree`、`to_degree` などを含む cyclic 角度 API 全体の命名を整理しました。

## フォーマット入出力とストリーム入出力

v1.0.0 では、xer 値型の入出力を強化しました。

`%@` フォーマットは、xer のフォーマット出力層から xer 値型を扱うための実用的な入口として強化されています。

また、次のような値型について iostream 演算子を追加・整理しました。

- `matrix`
- `vec`
- `polar`
- 色値型
- 画像ジオメトリ型
- cyclic / interval 値
- パスと物理量

これにより、診断出力、コード例、小さな確認用プログラムを書きやすくなりました。

新しいコード例では、`%@` フォーマットと通常の iostream 抽出演算子の使い方を示しています。

```text
examples/example_iostream_percent_at.cpp
```

## テストとコード例

v1.0.0 では、手編集や API 整理で壊れやすい箇所を中心にテストを追加・強化しました。

主な追加は次のとおりです。

- `tests/test_parse.cpp`
- `tests/test_version.cpp`
- `examples/example_parse_basic.cpp`
- `examples/example_color_conversion.cpp`
- `examples/example_cyclic_interval_concepts.cpp`

`tests/test_version.cpp` では、`<xer/version.h>` のプリプロセッサマクロと `xer` 名前空間の定数が一致していることを確認します。
また、`version_string` が `major.minor.patch + suffix` の形になっていることも確認します。

ドキュメントから参照されるコード例についても、実在するファイルと一致するよう確認・修正しました。

## 互換性メモ

### ソース互換性

v1.0.0 は、今後の安定版ベースラインとなるリリースです。

変更の多くは追加、整理、ドキュメント・テスト強化です。

注意が必要な主なソースレベルの整理は、次の古い別名の削除です。

```cpp
from_radian
to_radian
```

代わりに次を使用してください。

```cpp
from_rad
to_rad
```

### エラー処理

`error_t::success` の追加により、エラーコードの扱いがより明示的になりました。

これまで `static_cast<error_t>(0)` を名前のない成功値として扱っていたコードは、`error_t::success` を使う形に移行できます。
