# xer C++ Utility Library v0.7.0 Release Notes

xer C++ Utility Library v0.7.0 expands the numeric side of the library. This release adds public headers for practical real-number mathematics, descriptive statistics, and complex equation helpers, and aligns angle handling across math, matrix, cyclic, and image APIs around τrad units.

The release also updates the documentation set for the v0.7.0 API surface, including the public header list, reference manual fragments, Japanese reference fragments, and terminology guidance.

## Highlights

- Added `<xer/math.h>` for lightweight real-number mathematics, small vectors, polar coordinates, trigonometric helpers, vector helpers, and equation helpers.
- Added `<xer/statistics.h>` for descriptive statistics over arithmetic ranges.
- Added `<xer/complex.h>` for complex-root equation helpers.
- Standardized xer angle APIs around τrad units, where `1` means one full turn.
- Added degree/radian conversion helpers through the cyclic angle policy.
- Updated matrix rotation helpers to use cyclic τrad angles.
- Updated image arc APIs and documentation to use τrad angle semantics.
- Updated README files, public header lists, reference manual fragments, translation terminology, and policy documents for the v0.7.0 API set.

## New Public Headers

### `<xer/math.h>`

`<xer/math.h>` provides lightweight real-number mathematical helpers in xer. It is intended for practical formulas and small geometry-oriented calculations, not as a full numerical-analysis framework.

The main additions include:

- `vec<T, N>` for small 2D, 3D, and 4D vectors
- `polar<T, 2>` for polar coordinates using cyclic τrad angles
- trigonometric functions using τrad scalar values or `cyclic<T>` values
- inverse trigonometric functions returning τrad values
- polar/cartesian conversion helpers
- vector helpers such as `dot`, `length`, `distance`, `normalize`, `angle`, `rotate`, and `cross`
- `heron` for triangle area calculation
- `quadratic` and `cubic` for real-root equation solving

Angles in `<xer/math.h>` use τrad units. In this convention, `1` means one full turn, `0.25` means a right angle, and `0.5` means a half turn.

### `<xer/statistics.h>`

`<xer/statistics.h>` provides small descriptive statistical utility functions for arithmetic ranges.

The provided functions include:

- `sum`
- `product`
- `mean`
- `geometric_mean`
- `harmonic_mean`
- `median`
- `quantile`
- `percentile`
- `mode`
- `variance`
- `sample_variance`
- `stddev`
- `sample_stddev`

Most functions return `xer::result<double>` so that invalid input, such as an empty range or an invalid parameter, is reported explicitly. `mode` returns `xer::result<std::vector<double>>` because multiple values can share the highest frequency.

`mode` also accepts an optional tolerance value. This makes it useful for floating-point data where exact equality is often too strict.

### `<xer/complex.h>`

`<xer/complex.h>` provides complex-number counterparts for selected equation helpers.

The provided functions include:

- `cquadratic`
- `ccubic`

These functions accept real coefficients and return complex roots with multiplicity. They are separated from `<xer/math.h>` so that users of only real-number helpers do not need to include complex-number facilities.

## Angle Policy and τrad Unification

v0.7.0 standardizes angle handling around τrad units in the relevant xer APIs.

In τrad units:

- `1` means one full turn
- `0.5` means a half turn
- `0.25` means a right angle

This policy now applies consistently to:

- trigonometric functions in `<xer/math.h>`
- cyclic angle conversion helpers in `<xer/cyclic.h>`
- rotation helpers in `<xer/matrix.h>`
- arc drawing APIs in `<xer/image.h>`

Conversion helpers such as `from_degree`, `to_degree`, `from_rad`, and `to_rad` are provided for interaction with degree- and radian-based APIs.

## Matrix and Image Updates

`<xer/matrix.h>` now documents rotation helpers as cyclic τrad based operations. This includes 2D rotation and 3D rotations around the x, y, and z axes.

`<xer/image.h>` now documents circular and elliptical arc angles in τrad units. This keeps image drawing aligned with the rest of xer's angle policy and reduces unit-mixing mistakes.

## Compatibility Notes

v0.7.0 adds new public headers and functions. Existing v0.6.0 APIs remain within the documented v0.x stability policy.

The most important practical compatibility point is the angle-unit policy: newly documented math, matrix, cyclic, and image angle APIs use τrad unless explicitly documented otherwise. Code interacting with degree- or radian-based APIs should use the provided conversion helpers.
