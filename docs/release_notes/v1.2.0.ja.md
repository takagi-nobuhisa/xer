# xer C++ Utility Library v1.2.0 リリースノート

xer C++ Utility Library v1.2.0 は、モジュール連携機能の追加、診断APIの整理、テスト時間の短縮、および処理系間の移植性改善を行ったリリースです。

このリリースでは、共有ライブラリを実行時に読み込むための `<xer/dlfcn.h>` を新しい公開ヘッダーとして追加しました。また、従来の `<xer/diag.h>` を `<xer/diagnostics.h>` に改名し、簡易出力、トレース、ログを診断機能として整理しています。

公開ヘッダーの組合せテストは、すべての順序付き2ヘッダー組合せを既定でコンパイルする方式から、各公開ヘッダーを先頭にして残りの公開ヘッダーを続けてインクルードする方式へ変更しました。これにより、インクルード順序への依存を検査しながら、通常のテスト時間を大幅に短縮しています。

## Highlights

- 新しい公開ヘッダー `<xer/dlfcn.h>` を追加しました。
- `<xer/diag.h>` を `<xer/diagnostics.h>` に改名しました。
- `<xer/assert.h>` に `xer_assert_throw_any` を追加しました。
- `xer_assert_throw` が指定された例外型を `const` 参照で捕捉することを明確化しました。
- 公開ヘッダーのインクルード順序テストを高速化しました。
- 従来の全順序付きペアテストは `--full` または `--all-pairs` で実行できます。
- 上付き数字を含む非標準識別子をオプトイン化しました。
- 単位系に移植可能なASCII識別子 `m2`、`m3`、`sec2`、`sec3` を整備しました。
- README、公開ヘッダー一覧、方針文書、日英リファレンスマニュアルを更新しました。

## 新しい公開ヘッダー

### `<xer/dlfcn.h>`

共有ライブラリを実行時に読み込み、シンボルを取得するための小さなクロスプラットフォームAPIを追加しました。

主な型と関数は次のとおりです。

```cpp
auto dlopen(std::string_view path) -> result<shared_library>;
auto dlclose(shared_library& library) noexcept -> void;
auto is_open(const shared_library& library) noexcept -> bool;
auto native_handle(const shared_library& library) noexcept -> void*;

auto dlsym(
    const shared_library& library,
    std::string_view name,
    void*& out) -> result<void>;

template<class Function>
auto dlsym(
    const shared_library& library,
    std::string_view name,
    Function*& out) -> result<void>;
```

`shared_library`はコピー可能な共有ハンドルです。コピーされたオブジェクトは同じネイティブライブラリハンドルを共有し、最後の参照が解放されたときにライブラリをアンロードします。

POSIX環境では`dlopen`、`dlsym`、`dlclose`を、Windowsでは`LoadLibraryW`、`GetProcAddress`、`FreeLibrary`を内部で使用します。

関数ポインター用の`dlsym`オーバーロードでは、出力先のポインター型から関数型を推論するため、呼び出し側で明示的なキャストを書く必要がありません。

初期実装では、POSIX環境で`RTLD_NOW | RTLD_LOCAL`を使用し、Windowsでは`LoadLibraryW`を使用します。プラットフォーム固有のロードフラグはまだ公開していません。

## 診断ヘッダーの改名

従来の公開ヘッダー `<xer/diag.h>` を `<xer/diagnostics.h>` に改名しました。

新しいヘッダー名は、簡易出力、トレース、ログをまとめて提供する役割をより明確に表します。

主な機能は次のとおりです。

```cpp
xer_print(expression)
xer_print(label, expression)
xer_trace(category, level, object)
xer_log(category, level, message)
xer_log(category, level, format, ...)
```

`xer_trace`は`NDEBUG`が定義されている場合に無効化されます。`xer_log`は`NDEBUG`だけでは無効化されず、必要な場合は`XER_ENABLE_LOG`を`0`に定義してコンパイル時に無効化できます。

### 互換性上の注意

`<xer/diag.h>`は公開ヘッダーから削除されています。既存コードは次のように変更する必要があります。

```cpp
#include <xer/diagnostics.h>
```

## Assertion機能の追加

`<xer/assert.h>`に`xer_assert_throw_any`を追加しました。

```cpp
xer_assert_throw_any(expr)
```

このマクロは、式が何らかの例外を送出することを検査します。特定の例外型を要求しないテストや、標準例外型ではない値の送出を検査する場合に使用できます。

特定の例外型を検査する既存の`xer_assert_throw`は、指定された例外型を`const`参照で捕捉します。

```cpp
xer_assert_throw(expr, exception_type)
```

式が例外を送出しなかった場合、または指定した型と一致しない例外を送出した場合は、`xer::assertion_error`を送出します。

## 公開ヘッダーのインクルード順序テスト

`php/test_public_header_pairs.php`の既定テスト方式を変更しました。

従来は、公開ヘッダーが`n`個ある場合、異なる2ヘッダーのすべての順序付き組合せ`nP2`を生成してコンパイルしていました。

v1.2.0の既定方式では、公開ヘッダーごとに1つのテストソースを生成します。各ソースは対象ヘッダーを最初にインクルードし、その後に利用可能なすべての公開ヘッダーをインクルードします。

公開ヘッダーが47個の場合、既定方式でコンパイルする生成ソースは47件です。これにより、次の性質を検査しながらテスト時間を短縮します。

- 各公開ヘッダーを最初にインクルードできること
- 各公開ヘッダーが単独で必要な宣言を備えていること
- 後続する他の公開ヘッダーとの間にインクルード順序依存がないこと

従来の全順序付きペアテストも引き続き利用できます。

```sh
php test_public_header_pairs.php --full
php test_public_header_pairs.php --all-pairs
```

`--all`は、選択されたテスト方式に対して差分判定を無効化し、対象全体を再実行するオプションとして扱われます。

## 非標準識別子のオプトイン化

上付き数字など、標準C++の識別子として保証されない文字を含む別名を、既定では提供しない方針に変更しました。

単位系では、移植可能なASCII識別子として次を提供します。

```cpp
m2
m3
sec2
sec3
```

上付き数字を含む別名は、ヘッダーをインクルードする前に`XER_ENABLE_NON_STANDARD_IDENTIFIERS`を定義した場合のみ提供されます。

```cpp
#define XER_ENABLE_NON_STANDARD_IDENTIFIERS
#include <xer/quantity.h>
```

この場合、次の非標準別名も利用できます。

```cpp
m²
m³
sec²
sec³
```

同じ方針は、物理・数学定数の非標準Unicode識別子にも適用されます。

`-pedantic`などのオプションによる警告やエラーを避ける必要がある移植可能なコードでは、ASCII識別子を使用してください。

## モジュール連携方針

動的共有ライブラリ、遅延ロード、共有メモリ、同期プリミティブ、メッセージキュー、将来のRPCや言語バインディングを見据えたモジュール連携方針を追加しました。

v1.2.0では、その最初の独立した構成要素として`shared_library`と`<xer/dlfcn.h>`を実装しています。

C++向けの通常APIと、将来的に外部公開するC ABIは分離する方針です。C++ APIでは`xer::result<T>`、RAII、テンプレートなどを利用し、外部ABIではC互換型、状態コード、出力引数、不透明ハンドルを使用する方向を示しています。

## 互換性上の注意

v1.2.0には、次のソース互換性に影響する変更があります。

- `<xer/diag.h>`は `<xer/diagnostics.h>`へ改名されました。
- `m²`、`m³`、`sec²`、`sec³`などの非標準識別子は、`XER_ENABLE_NON_STANDARD_IDENTIFIERS`を定義しない限り提供されません。
- 移植可能な単位識別子として`m2`、`m3`、`sec2`、`sec3`を使用してください。

`<xer/dlfcn.h>`と`xer_assert_throw_any`は新規追加であり、既存APIを置き換えるものではありません。
