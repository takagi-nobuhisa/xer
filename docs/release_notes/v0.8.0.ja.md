# xer C++ Utility Library v0.8.0 リリースノート

xer C++ Utility Library v0.8.0 では、実用的な変換、ローカル通信支援、物理量の単位、診断出力の使いやすさを中心に拡張しました。このリリースでは、汎用変換入口 `xer::to<T>` を追加し、長さ付きソケットメッセージ送受信を一通り揃え、物理量の単位に国際ヤード・ポンド法の単位を追加し、コード例や簡単な確認に使いやすい軽量診断出力 `xer_print` を追加しています。

また、単位実装ヘッダの整理、コード例とテストの整理、v0.8.0 API に合わせたドキュメント更新も行っています。

## ハイライト

- 汎用変換関数 `xer::to<T>` を提供する `<xer/convert.h>` を追加
- `xer::in_range` に基づく範囲チェック付き数値変換を追加
- `char8_t`、`char16_t`、`char32_t`、`wchar_t` 文字列による明示的なテキスト変換に対応
- `xer::to<T>` では、曖昧な narrow `char` 文字列をテキストとして解釈しない方針を採用
- `<xer/socket.h>` に、アドレス指定 bind、完全送受信、長さ付きメッセージ送受信の補助関数を追加
- `<xer/quantity.h>` に国際ヤード・ポンド法の長さ・質量単位を追加
- 定義済み SI/common 単位とヤード・ポンド法単位を別々の内部実装ヘッダに分離
- `<xer/diag.h>` に、式や `result<T>` を簡単に表示する `xer_print` を追加
- コード例とテストで繰り返されていたローカル補助関数を整理
- README、公開ヘッダ一覧、リファレンスマニュアル断片、日本語リファレンス断片、翻訳用語表、方針文書を v0.8.0 API に合わせて更新

## 新しい公開ヘッダ

### `<xer/convert.h>`

`<xer/convert.h>` は、汎用変換関数 `xer::to<T>` を提供します。

```cpp
template <class To, class From>
auto to(From&& from) -> xer::result<To>;
```

`xer::to<T>` は C++ のキャスト式とは異なり、テキストの解析、値の文字列化、数値範囲の検証、一部の xer 値型の正規化を行う場合があり、失敗する可能性があります。そのため、戻り値は `xer::result<T>` です。

代表的な使用例は次のとおりです。

```cpp
auto n = xer::to<int>(u8"123");
auto s = xer::to<std::u8string>(123);
auto p = xer::to<xer::path>(u8"dir\\file.txt");
```

数値型同士の変換では、変換前に `xer::in_range` で範囲を確認します。これにより、意図しない縮小変換や signed/unsigned の範囲ミスを防ぎます。

```cpp
xer::to<unsigned char>(255); // 成功
xer::to<unsigned char>(256); // error_t::range
xer::to<unsigned>(-1);       // error_t::range
```

`char` は文字型として扱い、`signed char` と `unsigned char` は整数型として扱います。

`xer::to<T>` は、`char8_t`、`char16_t`、`char32_t`、`wchar_t` に基づく明示的に符号化された文字列を扱います。一方で、曖昧な narrow `char` 文字列はテキストとして解釈しません。そのため、`xer::to<int>("123")` や `xer::to<std::u8string>("abc")` のような呼び出しは拒否されます。これは、プラットフォーム依存の narrow 文字列を UTF-8、ロケール文字列、ネイティブパス文字列のいずれかとして暗黙に扱うことを避けるためです。

## ソケット機能の更新

`<xer/socket.h>` には、小規模な TCP ベースのローカルプロトコルなどで使いやすい補助関数を追加しました。

新規または拡張された主な操作は次のとおりです。

- `127.0.0.1` など特定のローカルアドレスに bind する `socket_bind(socket&, address, port)`
- ストリームソケットにバイト列全体を送る `socket_send_all`
- ストリームソケットから指定バイト数を正確に受け取る `socket_recv_exact`
- 4バイト big-endian 長さ付きメッセージを送る `socket_send_message`
- 呼び出し側が指定した最大サイズ以内で長さ付きメッセージを受け取る `socket_recv_message`

長さ付きメッセージの形式は次のとおりです。

```text
uint32 big-endian payload_size
payload bytes
```

`socket_recv_message` は、指定された最大サイズを超えるメッセージを拒否し、その oversized payload は読み捨てません。このようなプロトコルエラーの後は、呼び出し側が接続を閉じる想定です。

これらの補助関数は、HTTP/PHP API 層と常駐制御プロセスが TCP localhost で通信するような構成で特に有用です。

## 物理量と単位の更新

`<xer/quantity.h>` では、既存の SI 単位・慣用的なメートル法単位に加えて、国際ヤード・ポンド法の単位を追加しました。

追加した単位オブジェクトは次のとおりです。

- `inch`
- `ft`
- `yd`
- `mile`
- `oz`
- `lb`

これらは `xer::units` 名前空間で提供され、該当する単位は SI 単位に対する国際定義に基づいています。

- `1 inch = 0.0254 m`
- `1 ft = 0.3048 m`
- `1 yd = 0.9144 m`
- `1 mile = 1609.344 m`
- `1 lb = 0.45359237 kg`
- `1 oz = 1/16 lb`

公開ヘッダとしての使い方は従来どおりです。

```cpp
#include <xer/quantity.h>
```

内部的には、定義済み単位を `xer/bits/units_si.h` や `xer/bits/units_imperial.h` などの実装ヘッダに分離しました。これは実装上の整理であり、これらの `xer/bits/` 配下のヘッダを公開ヘッダとして扱うものではありません。

## 診断出力

`<xer/diag.h>` に、簡単な診断出力用の `xer_print` を追加しました。

```cpp
xer_print(expression);
xer_print(label, expression);
```

1引数形式では、ソース上の式文字列をラベルとして使います。2引数形式では、明示的な UTF-8 ラベルを使います。

`xer_print` は、xer の `%@` printf 変換を使って値を整形します。`result<T>` については、成功時は格納値を表示し、失敗時は `error(name)` を表示します。`result<void>` の成功時は `ok` を表示します。

`xer_print` は、コード例、簡単な確認、軽量な診断出力のための機能です。カテゴリとレベルを持つトレース機能である `xer_trace` とは別の役割を持ちます。`xer_trace` は従来どおり、`NDEBUG` 定義時には無効化されます。

## コード例とテストの整理

コード例では、単純な表示のために各ファイルで `print_result`、`print_line` などのローカル補助関数を繰り返し定義する代わりに、`xer_print` を使うよう整理しました。

テストでは、ファイル、バイト列、一時ディレクトリなどの操作に関する共通のテスト補助を整理しました。これにより、テスト対象の挙動に集中しやすくしています。

これらはリポジトリ内のコード例・テストの整理であり、公開 API には影響しません。

## 互換性メモ

v0.8.0 は、新しい公開ヘッダ1件と複数の関数を追加するリリースです。既存の v0.7.0 公開 API は、文書化されている v0.x の安定性方針の範囲にあります。

実用上もっとも重要な挙動上の注意点は、`xer::to<T>` が narrow `char` 文字列をテキストとして解釈しないことです。テキストの解析や変換を行う場合は、`u8"..."`、`u"..."`、`U"..."`、`L"..."` など、明示的に符号化された文字列リテラルを使ってください。

単位定義の内部ヘッダは保守性のために分離されましたが、利用者は従来どおり `<xer/quantity.h>` を include してください。`xer/bits/` 配下のヘッダを直接 include することは想定していません。
