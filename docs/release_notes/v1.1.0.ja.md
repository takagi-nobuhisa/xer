# xer C++ Utility Library v1.1.0 リリースノート

xer C++ Utility Library v1.1.0 は、対応 toolchain とテスト基盤の強化を中心としたリリースです。

このリリースでは、新しい大きな機能領域は追加していません。その代わりに、Clang 系 toolchain と Visual Studio 2026 環境への対応を拡充し、GCC、Clang、MSYS2 CLANG64、Visual Studio clang-cl、Visual Studio MSVC cl.exe の各環境で、同じテストプログラム、コード例、公開ヘッダーペア確認を実行できるようにしました。

## ハイライト

- Ubuntu Clang + libc++ に対応
- MSYS2 CLANG64 に対応
- Visual Studio 2026 clang-cl に対応
- Visual Studio 2026 MSVC cl.exe に対応
- `php/run_tests.php` を更新し、GCC、Clang、clang64、clang-cl、MSVC を明示的に切り替えられるように整理
- `php/test_public_header_pairs.php` を更新し、同じ toolchain 選択モデルに対応
- Visual Studio 2026 向けに、vcpkg manifest mode による ICU / zlib 検出に対応
- Tcl/Tk の Windows 向け検出を強化し、Tcl/Tk 9.0 を 8.6 より優先
- Windows API、Winsock2、`NOMINMAX`、`min` / `max` マクロまわりを整理
- Windows のディレクトリ列挙を `FindFirstFileExW` / `FindNextFileW` ベースに変更
- MSVC 向けに `/Zc:__cplusplus` と `/Zc:preprocessor` を扱うように整理
- `__int128` を通常機能として使わない環境への対応を整理
- テスト失敗時に未捕捉例外の情報を `run.log` に出力するように改善
- MSVC / clang-cl のオブジェクトファイルを `php/build/` 配下に出力するように整理
- `vcpkg_installed/` を Git 管理およびリリース ZIP の対象外に整理
- README、方針文書、リファレンス断片を対応 toolchain 拡充に合わせて更新

## 対応 toolchain の拡充

v1.1.0 では、公式に対応・検証する toolchain を拡充しました。

現在のコンパイラ対象は次のとおりです。

- GCC 13.3.0 以降
- Clang 18.0.0 以降
- Visual Studio 2026 clang-cl
- Visual Studio 2026 MSVC cl.exe

検証対象環境は次のとおりです。

- Ubuntu + GCC
- Ubuntu + Clang + libc++
- MSYS2 UCRT64 + GCC
- MSYS2 CLANG64 + Clang
- Visual Studio 2026 + clang-cl
- Visual Studio 2026 + MSVC cl.exe

MSVC cl.exe 対応は、他の対応 toolchain と同じテストおよびコード例の範囲で確認しています。

MSVC cl.exe を直接使用する場合、xer では次の標準準拠オプションが必要です。

```bat
/Zc:__cplusplus /Zc:preprocessor
```

`/Zc:__cplusplus` は、`__cplusplus` が選択中の C++ 標準モードを正しく返すために必要です。
`/Zc:preprocessor` は、xer の診断系マクロで使用している可変引数マクロを標準準拠で展開するために必要です。

付属の PHP テストスクリプトでは、`--tc=msvc` 指定時にこれらのオプションが自動的に追加されます。

## テストスクリプトとビルド基盤の更新

`php/run_tests.php` と `php/test_public_header_pairs.php` を更新し、複数 toolchain を明示的に切り替えて検証できるようにしました。

代表的な実行例は次のとおりです。

```sh
php run_tests.php tests examples --jobs=8 --all --tc=gcc
php run_tests.php tests examples --jobs=8 --all --tc=clang
php run_tests.php tests examples --jobs=8 --all --tc=clang64
php run_tests.php tests examples --jobs=8 --all --tc=clang-cl
php run_tests.php tests examples --jobs=8 --all --tc=msvc
```

公開ヘッダーペア確認も toolchain 指定に対応しています。

```sh
php test_public_header_pairs.php --jobs=8 --tc=gcc
php test_public_header_pairs.php --jobs=8 --tc=clang
php test_public_header_pairs.php --jobs=8 --tc=clang64
php test_public_header_pairs.php --jobs=8 --tc=clang-cl
php test_public_header_pairs.php --jobs=8 --tc=msvc
```

ビルド出力は toolchain ごとに `php/build/` 配下へ分離されます。

```text
php/build/linux-gcc/
php/build/linux-clang/
php/build/windows-msys2-ucrt64/
php/build/windows-msys2-clang64/
php/build/windows-msvc-clang-cl/
php/build/windows-msvc-cl/
```

MSVC と clang-cl が生成するオブジェクトファイルも、`php` ディレクトリ直下ではなく、対応するビルドディレクトリ配下へ出力されます。

## Windows / Visual Studio 2026 向け外部依存

v1.1.0 では、Visual Studio 2026 環境で任意外部依存を扱えるようにしました。

リポジトリには `vcpkg.json` が含まれています。ICU と zlib は、プロジェクトルートで次を実行することでインストールできます。

```bat
vcpkg install --triplet x64-windows
```

vcpkg manifest mode では、依存ライブラリは次の下にインストールされます。

```text
vcpkg_installed\x64-windows
```

`run_tests.php` は、この配下の include、lib、bin ディレクトリを検出し、コンパイル時のインクルードパス、リンクライブラリ、実行時 DLL 探索パスへ反映します。

Tcl/Tk についても Windows 向け検出を強化しました。Tcl/Tk 9.0 を 8.6 より優先します。自動検出で不足する場合は、`XER_TEST_TCLTK_ROOT` でインストール先を指定できます。

```bat
set XER_TEST_TCLTK_ROOT=C:\local\IronTcl
```

Magicsplat Tcl/Tk、ActiveTcl、その他の Windows 向けバイナリ配布は、必要なヘッダー、インポートライブラリ、実行時 DLL を提供していれば利用できます。

## Windows API と Winsock2 まわりの整理

Visual Studio 2026 対応に合わせて、Windows ヘッダーの扱いを整理しました。

主な変更は次のとおりです。

- `winsock2.h` と `windows.h` の include 順序を整理
- `NOMINMAX` 対応を追加
- `min` / `max` マクロ衝突への対策を追加
- 通常の Windows API 利用とソケット固有の Windows ヘッダー利用を分離
- Windows のディレクトリ列挙を `FindFirstFileExW` / `FindNextFileW` ベースに変更

これにより、`<xer/socket.h>` と他の Windows 系ヘッダーを組み合わせた場合の衝突を抑えています。

## `__int128` 非対応環境への対応

MSVC および MSVC 互換環境では、`__int128` を xer の通常機能としては使用しない方針に整理しました。

`XER_HAS_INT128` を導入し、`__int128` を安全に使用できる環境だけで 128 ビット整数経路を有効にします。MSVC および clang-cl 環境では、必要に応じて 64 ビット整数ベースの処理へフォールバックします。

---

## MSVC で使用できない識別子への対応

MSVC cl.exe では、識別子中の上付き数字などを受け付けません。

そのため、影響を受ける単位系の識別子に ASCII 形式の別名を追加しました。

```cpp
m2
m3
sec2
sec3
```

MSVC ではこれらの別名を使用します。記号的な識別子を受け付ける toolchain では、利用可能な範囲で従来の記法も使用できます。

## テスト診断の改善

テストプロセスが未捕捉例外によって終了した場合、理由を `run.log` に出力するようにしました。

これにより、MSVC 環境のように assertion failure が実行時の異常終了として見える場合でも、原因を特定しやすくなっています。

---

## パッケージングの整理

vcpkg manifest mode によって生成される `vcpkg_installed/` は、Git 管理およびリリース ZIP の対象外としました。

`php/build/`、オブジェクトファイル、ログ、実行ファイル、DLL、インポートライブラリ、一時ファイルなどの生成物も、リリース ZIP には含めない方針です。
