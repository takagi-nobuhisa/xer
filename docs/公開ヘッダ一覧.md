# 公開ヘッダ一覧

## 基本方針

- 公開ヘッダは `xer/` 直下に配置する。
- `xer/bits/` 配下のヘッダは内部実装用であり、公開ヘッダとはみなさない。
- C 標準ライブラリのヘッダ名をそのまま揃えることは目的とせず、XER として意味のある単位で再構成する。
- `wchar.h` および `wctype.h` は独立ヘッダとしては提供せず、関連機能は他ヘッダに吸収する。
- `math.h` は将来検討とし、現時点では公開ヘッダに含めない。

---

## 提供する公開ヘッダ

### 基盤

- `xer/error.h`
- `xer/assert.h`

### 文字・文字列処理

- `xer/string.h`
- `xer/ctype.h`
- `xer/stdlib.h`

### 入出力・ファイル関連

- `xer/stdio.h`
- `xer/path.h`

### 数値・演算

- `xer/stdint.h`
- `xer/arithmetic.h`

### 時刻

- `xer/time.h`

### バージョン情報

- `xer/version.h`

---

## 独立ヘッダとしては提供しないもの

以下は独立した公開ヘッダは設けず、必要な機能を既存ヘッダに吸収する。

- `wchar.h`
  - 主に `stdlib.h`, `stdio.h`, `time.h` で吸収する
- `wctype.h`
  - 主に `ctype.h` で吸収する

---

## 現時点で再実装対象外とするヘッダ

以下のヘッダは、少なくとも現段階では XER として再実装しない。

- `complex.h`
- `fenv.h`
- `float.h`
- `limits.h`
- `locale.h`
- `signal.h`

## 補足

### `locale.h` を提供しない理由

XER はロケール依存を極力排し、文字種判定・文字変換・文字コード変換をライブラリ独自方針で扱うため、`locale.h` は公開 API の中核に置かない。

### `wchar.h` / `wctype.h` を分離しない理由

XER では C 標準ライブラリのヘッダ構成をそのまま模倣するのではなく、用途ごとに API を再配置する方針を取る。  
そのため、`wchar_t` 系・ワイド文字分類系の機能は、以下のように既存ヘッダへ統合する。

- 文字変換: `stdlib.h`
- テキスト入出力: `stdio.h`
- 文字分類・文字変換: `ctype.h`
- 時刻文字列化の関連部分: `time.h`
